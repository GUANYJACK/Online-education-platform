# 网页端界面设计与功能需求（MVP）

## 1. 文档依据
- `CLAUDE.md`：遵循MVP优先、演示优先、简单可串联原则。
- `Jobs Duty Assignment.md`：网页端聚焦教师端与学校管理员端主流程、数据看板与页面结构。
- `PRD.md`：以教师端/学校端用户故事与验收标准为功能基线。

## 2. 设计目标与范围
- 目标：在 React 网页端完成"教师班级管理 + 学校数据看板"的可演示原型。
- 范围：教师端、学校管理员端、登录认证、角色区分、班级数据监控、心理健康占位页。
- 非范围（MVP不做或仅占位）：
  - 数据导出真实功能（仅预留按钮）
  - 心理健康真实分析（仅占位UI与空数据态）
  - 复杂交互（拖拽、批量操作等）

## 3. 网页端信息架构

### 3.1 全局结构
- 登录页（邮箱+密码）
- 注册页
- 加入学校/组织页
- 主框架（侧边导航 + 内容区）

### 3.2 教师端导航（侧边栏）
1. 班级总览：所教班级列表、核心指标
2. 班级详情：学生列表、知识点掌握率
3. 学生管理：查看/移除学生
4. 心理健康：班级心理健康聚合报告（占位）
5. 个人设置：账号信息、退出登录

### 3.3 学校管理员端导航（侧边栏）
1. 全校总览：全校核心指标仪表盘
2. 年级视图：按年级汇总数据
3. 班级视图：全校班级列表与数据
4. 教师管理：教师列表
5. 心理健康：全校心理健康数据看板（占位）
6. 个人设置：账号信息、退出登录

## 4. 界面设计要求

### 4.1 统一设计规范
- 风格：简洁、企业级后台、数据可读性优先。
- 颜色语义：
  - 已掌握：绿色
  - 部分掌握：橙色
  - 未掌握：灰/红
  - 心理风险：低/中/高使用绿/黄/红
- 通用组件：
  - 数据卡片、统计数字、进度条、表格、趋势图容器、空状态组件。

### 4.2 关键页面（教师端）

1. 登录/注册页
   - 邮箱+密码登录。
   - 注册后引导进入"加入学校/组织"页面。

2. 班级总览页
   - 展示所教班级列表（班级名称、学生人数、平均掌握率、平均学习进度）。
   - 每个班级卡片可点击进入班级详情。
   - 支持按知识点筛选查看（哪些学生未掌握某知识点）。

3. 班级详情页
   - 学生列表：姓名、账号、各知识点掌握率。
   - 支持按知识点筛选。
   - 点击学生可查看详细学习数据（弹窗或跳转）。
   - 移除学生功能。

4. 心理健康页（占位）
   - 班级整体心理健康聚合指标（情绪极性分布、平均风险等级）。
   - 学生列表展示各学生风险等级（低/中/高）。
   - 支持按风险等级筛选。
   - 高风险学生醒目提示（红色标签）。
   - 默认空数据态与"后续接入AI分析"提示。

### 4.3 关键页面（学校管理员端）

1. 全校总览仪表盘
   - 核心指标卡片：学生总数、班级总数、平均掌握率、平均学习进度。
   - 年级维度汇总数据。
   - 全校心理健康聚合指标（占位）。

2. 年级视图
   - 按年级展示各班数据对比。
   - 支持下钻到班级详情。

3. 班级视图
   - 全校班级列表，支持按年级/教师筛选。
   - 展示各班核心指标。

4. 心理健康看板（占位）
   - 全校/全年级心理健康聚合指标。
   - 情绪极性分布、风险等级分布、压力关键词云。
   - 全校心理健康趋势图表（时间轴+各风险等级人数变化）。
   - 支持按年级、班级维度下钻。
   - 默认空数据态。

## 5. 功能需求（按优先级）

### 5.1 P0（必须完成）
- 角色登录（邮箱+密码）
- 教师班级列表与核心指标展示
- 班级学生列表与知识点掌握率
- 学生管理（查看/移除）
- 学校管理员全校数据仪表盘
- 年级/班级维度数据下钻
- 角色权限隔离（教师仅看自己班级，管理员看全校）

### 5.2 P1（应完成/可占位）
- 心理健康页面UI与空状态（教师端+管理员端）
- 数据导出按钮占位（提示"即将上线"）
- 加入学校/组织流程

### 5.3 暂缓项（MVP后）
- 数据导出真实功能
- 心理健康AI分析与高风险告警联动
- 教师管理工具（批量操作等）

## 6. 关键业务规则
- 掌握度分级：未掌握 / 部分掌握 / 已掌握。
- 教师权限边界：仅可查看自己班级数据，无法查看学生与AI的对话内容。
- 管理员权限：可查看全校/全年级数据，可手动编辑学生知识点掌握等级。
- 心理健康数据边界：仅展示聚合结果，不展示原始情绪数据。

## 7. 页面联动与演示主流程

### 7.1 教师端演示链路
登录 → 加入学校 → 查看班级总览 → 进入班级详情 → 查看学生掌握率 → 查看心理健康占位页。

### 7.2 学校管理员端演示链路
登录 → 查看全校仪表盘 → 按年级下钻 → 查看班级数据 → 查看全校心理健康占位页。

### 7.3 角色切换
登录后根据角色自动加载对应导航与数据视图。管理员可切换到教师视图查看特定班级。

## 8. 数据展示与空状态要求
- 所有图表支持空态：显示"暂无数据"与引导文案。
- 心理健康模块默认空态，不报错、不崩溃。
- 接口失败时展示轻提示并允许重试，不阻断主流程演示。

## 9. 验收口径（网页端）
- 主流程可完整演示，页面可串联、跳转正确。
- 教师端与管理员端核心页面齐备，角色边界清晰。
- 文案、结构与PRD用户故事一致，不扩展超出MVP范围。
- 满足课堂展示目标：叙事清楚、页面完整、数据口径统一。
